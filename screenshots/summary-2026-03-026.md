### 5. Sync Implementation

**`PrdbSyncService`** — scoped service implementing the full sync flow:

1. Loads API key + URL from `AppSettings` (aborts gracefully if not configured)
2. Fetches all ~526 sites (paginated) → upserts `PrdbSite` + derives `PrdbNetwork`
3. Fetches latest 1,500 global videos → upserts `PrdbVideo`
4. Fetches all videos for each favorite site → upserts `PrdbVideo` (deduplicated)

All operations are **upsert by prdb GUID** — safe to re-run. Updates are batched in chunks of 200 to avoid large queries.

**`POST /api/prdb-sync`** — manual trigger, returns `{ networksUpserted, sitesUpserted, videosUpserted }`

**Design notes:**
- Video detail (pre-names, actors, images) is intentionally **not** fetched yet — the detail endpoint costs one request per video
- The existing `SyncWorker` background service is already wired up but its `RunAsync` is still a TODO — `PrdbSyncService` is injectable and ready to be called from it

2. **Actor sync** — fetch actors appearing in synced videos via `GET /actors/{id}`. Also needs batching. Currently `PrdbActor` table is empty.

3. **Favorite sites management** — the `IsFavorite` / `FavoritedAtUtc` fields on `PrdbSite` are populated by the prdb API sync (from `/favorite-sites`), but the current sync doesn't call that endpoint yet. Need to add a step that fetches the user's favorite sites and marks them in the DB.

4. **Commit and push the sync work** — the sync implementation from this session hasn't been committed yet.

### Medium priority

5. **Wire `PrdbSyncService` into `SyncWorker`** — the background service runs every 15 minutes and has a TODO. One call to `syncService.SyncAsync(ct)` is all that's needed once scheduled sync is wanted.

6. **Rate limit awareness** — the sync makes no attempt to respect the 1,000 req/hour limit. For video detail sync (potentially thousands of requests) this will matter.

### Lower priority

10. **Sync progress feedback** — sync can take a while (hundreds of paginated requests). A progress indicator or server-sent events would improve UX over the current "button spins until done" approach.
